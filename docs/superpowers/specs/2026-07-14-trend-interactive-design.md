# 지원경향 탭 인터랙티브화 — 설계

- 작성일: 2026-07-14
- 대상: 수시 NAVI 웹앱 지원경향 탭

## 문제
지원경향 탭이 원본 엑셀에 설정돼 있던 기준 1개(아주대/종합/ACE/경영인텔리전스학과)만 고정 스냅샷(`trend.json`)으로 표시. 기준을 바꿀 수 없다.

## 해결
기준을 선택하면 해당 기준의 지원경향(어디에 같이 지원했고 합격률이 얼마인지)을 보여준다. 이 데이터는 이미 `public/data/hapbul/byUnit_*.json`(모집단위별, 기준 대학별 청크)에 전부 있음 — 재사용한다.

## 데이터
`byUnit_*` 청크 컬럼: 기준_대학, 기준_대전형, 기준_전형, 기준_계열, 기준_모집단위, 지역, 대학, 대전형, 전형, 계열, 모집단위, 모집인원, 지원건수, 합격, 합격률. (컬럼 인덱스는 이름으로 조회 — 하드코딩 금지)

## UI (신규 `TrendTab.tsx`, 'trend' 키를 SpecialTab 대신 이 컴포넌트로 라우팅)
1. 기준 대학 선택(검색 가능, `byUnit_index.json`의 bases ~231개). HapbulTab의 선택 UI 재사용/참고.
2. 선택 시 해당 청크 로드(캐시). 청크에서 파생한 값으로 종속(cascading) 선택: 기준_대전형 → 기준_전형 → 기준_계열 → 기준_모집단위. 각 선택지는 상위 선택으로 필터된 청크의 distinct 값.
3. 선택된 기준(부분 선택 허용)에 매칭되는 행을 필터 → 결과표에 타지원 컬럼(지역/대학/대전형/전형/계열/모집단위/모집인원/지원건수/합격/합격률) 표시. 검색 + 페이지네이션(기존 표 컴포넌트/스타일 재사용).
4. 상단에 "기준: 대학 / 대전형 / 전형 / 계열 / 모집단위" 표시.

## 비범위
- 원본의 30/50/70%컷 통계도표는 포함하지 않음(별도 데이터). 표 기반 지원경향만.
- 기존 정적 `trend.json`/`_special_trend`는 제거하지 않아도 됨(사용 안 하면 그만). App 라우팅만 TrendTab으로 변경.
- 엔진·다른 탭 불변.
