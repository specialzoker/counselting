# 교과·종합전형 판정 — 설계 문서

- 작성일: 2026-07-13
- 대상: 수시 NAVI 웹앱 (판정 탭 결과표)

## 목표

현재 판정은 정시 백분위 차이값 하나뿐이다. 이미 입력받는 학생 내신을 각 대학의 교과1/2/3·종합1/2 컷(내신 등급)과 비교해 **교과·종합전형도 안정/적정/소신/도전으로 판정**한다.

## 설계

### ① 내신 기준 선택
결과표 상단에 "내신 기준" 셀렉터(전교과·국수영사과·국수영사·국수영과, 기본 전교과). 선택한 조합의 학생 내신 평균(`comboAverage`, 기존 함수)을 판정 기준값으로 사용.

### ② 판정 계산 (기존 50%·70% 컷 활용)
각 교과1/2/3·종합1/2 전형에 대해 학생 내신 등급 `n`을 비교:
- **안정**: `n <= p50`
- **적정**: `p50 < n <= p70`
- **소신**: `p70 < n <= p70 + margin`
- **도전**: `n > p70 + margin`
- 컷(p50/p70)이 없거나 학생 내신이 없으면 라벨 없음(null).

판정은 항상 **9등급 컷(p50/p70)** 기준(학생 내신이 9등급이므로). 표의 5등급 토글은 표시만 바꾸고 판정에는 영향 없음. `margin` 기본 0.5등급, `config.ts`에서 조정 가능.

### ③ 표시
각 교과1/2/3·종합1/2 셀에 안정/적정/소신/도전 색 라벨(정시 판정과 동일 색 체계)을 컴팩트하게(컷 숫자 옆 작은 색 배지). 내신 미입력 시 라벨 없음.

### ④ 구현 단위
- `src/engine/gyogwaJudge.ts`: 순수함수 `gyogwaLabel(naeshin: number|null, p50: number|null, p70: number|null, margin: number): JudgeLabel|null`. 단위테스트.
- `src/config.ts`: `GYOGWA_MARGIN = 0.5`.
- `src/ui/ResultTable.tsx`: 내신 기준 셀렉터 + 각 전형 셀 라벨. 정시 판정 엔진(judge.ts/percentile.ts)·다른 탭 불변.
- 내신 기준값: App에서 `naeshin`(기존 상태)로부터 선택 조합의 `comboAverage`를 계산해 ResultTable에 전달.

## 비범위
- 전형별 반영교과 자동 매핑(교과반영 탭 연동)은 하지 않음 — 상담자가 조합을 선택.
- 정시 백분위 차이값·순위 로직은 변경하지 않음.
