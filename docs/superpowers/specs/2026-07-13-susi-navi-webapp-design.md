# 수시 NAVI 웹앱 — 설계 문서 (1단계: 핵심 판정 엔진)

- 작성일: 2026-07-13
- 원본: `경기도교육청_2027수시NAVI_수시나비__260707_보호해제.xlsx` (30개 탭)
- 대상 사용자: 학원·컨설팅 종사자 (진로진학 상담)
- 배포: GitHub Pages (정적 사이트)

## 1. 목표

엑셀 파일의 "학생 성적 입력 → 대학 판정 + 정보 제공" 기능을 웹앱으로 재현한다.
학원·컨설팅용이므로 **원본 엑셀의 판정 계산 결과와 값이 일치**하는 것을 최우선으로 한다.

30개 탭 전체를 한 번에 만들지 않고 단계적으로 구축한다.
- **1단계 (이 문서 범위):** 핵심 판정 엔진 — `검색`, `내신`, 모의고사(3~11월), `점수계산기`
- 2단계 이후: 참고 데이터 탭(전형별, 모집단위별, 종합, 논술, 특별전형, 교과반영, 수능최저, 전공자율, 지원경향, 사례차트, 전형일정, 2028대입, 백분위조견표, 등급변환표 등)

## 2. 비범위 (1단계에서 하지 않는 것)

- 로그인 / 계정 / 서버 저장 (정적 앱, 브라우저에서만 동작)
- 12만 행 규모 대용량 조회 탭(`전형별`, `모집단위별`)의 전체 재현 — 2단계로 미룸
- 원본 엑셀 편집/저장 기능

## 3. 아키텍처

### 3.1 방식: 정적 웹앱 (A안)
- 엑셀 데이터를 1회성 스크립트로 JSON으로 추출 → 앱에 내장
- 판정 계산은 전부 브라우저(클라이언트) 안에서 수행
- 서버·DB 없음 → GitHub Pages로 무료 배포

### 3.2 기술 스택
- React + Vite + TypeScript
- 스타일: (구현 단계에서 확정, 경량 CSS 또는 Tailwind)
- 배포: GitHub Pages (Vite `base` 경로 설정)

### 3.3 데이터 파이프라인
```
[원본 .xlsx]
   │  Python(openpyxl, data_only=True) 추출 스크립트 (scripts/extract.py)
   ▼
[public/data/*.json]   ← 앱에 포함, 커밋
   │  fetch()
   ▼
[판정 엔진 (TypeScript)]
```
- 검증 완료: `data` 시트 6,368개 모집단위의 컷 백분위(교과1/2/3·종합1/2의 50%·70% 값, 5등급 변환값 포함)가 캐시된 값으로 전부 추출 가능함을 확인함.
- 추출 스크립트는 원본 파일 경로를 인자로 받아 재실행 가능하게 한다(원본 갱신 시 데이터만 다시 뽑음).

## 4. 판정 엔진 (원본 `검색` 로직 이식)

원본 엑셀 수식(INDEX/MATCH/RANK/IFERROR)을 TypeScript로 그대로 옮긴다.

### 4.1 입력
- 내신: 학기별(1-1, 1-2, 2-1, 2-2, 3-1) × 교과유형(전교과, 국수영사과, 국수영사, 국수영과 등) 등급
- 모의고사 백분위: 국어, 수학, 탐구1, 탐구2, 영어, 한국사
- 기준 모의고사 월: 3, 5, 6, 7, 9, 10, 11월 중 선택
- 필터: 지역(서울/경인/충청/강원/경상/전라/제주), 계열(공통/인문/자연/의약학/예체능)
- 5등급제 변환 토글 (원본 `검색!V6` 스위치 → `data` 시트의 5등급 컬럼 사용)

### 4.2 계산 (원본 그대로)
1. 모의고사 평균 지표
   - 국수탐(1) = AVERAGE(국, 수, MAX(탐1, 탐2))
   - 국수탐(2) = AVERAGE(국, 수, AVERAGE(탐1, 탐2))
2. 각 모집단위에 대해:
   - 학생 백분위(원본 `data` 시트 학생기준 컬럼)와 대학 **70% 컷 백분위**를 가져옴
   - **차이값 = 학생 백분위 − 70% 컷**
3. 차이값 기준으로 순위 정렬 (원본 `RANK` + 동점 처리 로직 그대로)
4. 각 전형(교과1/2/3, 종합1/2, 논술)의 50%·70% 컷 표시

### 4.3 출력
결과 표에 모집단위별로 표시:
- 기본: 지역, 세부지역, 대학, 27수시/26수시 모집단위, (정시)전형명
- 반영영역, 학생 백분위, 70% 백분위, 영어/한국사, **차이값**
- 교과1/2/3·종합1/2·논술 각 전형명 + 50%/70% 컷
- 정렬/필터, 5등급제 변환 토글

### 4.4 판정 라벨 (원본에 없는 추가 기능)
차이값을 기준으로 색깔 라벨을 보조 표시한다. 원본의 차이값·순위 숫자는 그대로 두고 그 위에 얹는다.
- 라벨: 안정 / 적정 / 소신 / 도전
- 경계값(차이값 기준)은 **설정 파일의 조정 가능한 상수**로 분리한다. 기본값(가안, 추후 조정):
  - 안정: 차이값 ≥ +5
  - 적정: 0 ≤ 차이값 < +5
  - 소신: −3 ≤ 차이값 < 0
  - 도전: 차이값 < −3
- (경계값 확정은 구현 후 사용자와 함께 조정)

## 5. 정확도 검증 (필수)

학원용이므로 이식한 계산이 원본과 일치하는지 반드시 검증한다.
- 여러 학생 케이스(다양한 내신·백분위·지역·계열)를 정하고
- 원본 엑셀의 `검색` 결과 표와 웹앱 출력의 차이값·순위·컷 값을 나란히 비교
- 불일치 시 수식 이식 오류를 수정 (systematic-debugging)

## 6. 화면 구성

원본 `검색` 탭 배치를 웹으로 재구성 (목업으로 방향 승인됨):
- 상단: 학생 성적 입력 패널 (내신 요약, 모의고사 백분위, 기준 월, 지역·계열 필터, 5등급제 토글)
- 하단: 판정 결과 표 (정렬·필터, 판정 라벨)

## 7. 위험 요소 및 대응

- **판정 로직 정확도** → 검증 단계로 방어 (5절)
- **원본 데이터 저작권/출처(경기도교육청)** → GitHub Pages 공개 저장소로 배포 시 원본 데이터가 공개됨. 출처 표기 및 공개 여부는 사용자 판단. (구현과 무관, 배포 전 확인)
- **원본 수식의 배열수식/INDIRECT 등 복잡 참조** → 추출·이식 시 개별 확인 필요
