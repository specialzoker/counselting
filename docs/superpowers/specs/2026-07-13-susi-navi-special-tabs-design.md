# 수시 NAVI — 남은 특수 탭 4종 (정적 표) 설계

## 목표

원본 `2027수시NAVI` 엑셀의 아직 웹앱에 없는 특수 시트 4종(**안내필독·백분위조견표·등급변환표·지원경향**)을, 엑셀의 인터랙티브 동작은 재현하지 않고 **원본 데이터를 그대로 보여주는 정적 표/텍스트 탭**으로 추가한다. 기존 참고 탭(RefTable) 수준의 깔끔함을 유지한다.

## 배경 / 현재 상태

- 이미 라이브인 탭: 판정(엔진), 참고 8탭(RefTable), 합불사례 2탭(HapbulTab).
- 기존 범용 추출기 `extract_refs()` + `RefTable`은 "헤더행 + 연속 데이터"인 정형 시트만 처리한다.
- 이번 4개 시트는 불규칙(텍스트 문단, 가로 다중 패널, 다중 섹션, 좌우 병렬 표)이라 범용 추출기로는 지저분해진다 → **시트별 맞춤 추출**이 필요.

## 접근 (승인된 방식 A)

1. `scripts/extract.py`에 시트별 맞춤 추출 함수를 추가해, 의미 있는 하위 표만 뽑아 통일 스키마 JSON으로 저장한다.
2. RefTable의 표 렌더링을 프레젠테이션 컴포넌트 `DataTable`로 분리한다.
3. 새 컴포넌트 `SpecialTab`이 `intro` 문단 + 각 `table`을 `DataTable`로 스택 렌더한다. 4개 탭 모두 `SpecialTab`을 재사용한다.

## 데이터 스키마

각 특수 탭은 `public/data/special/<key>.json` 한 파일:

```json
{
  "sheet": "백분위조견표",
  "key": "johgyeon",
  "intro": ["문단1", "문단2", ...],
  "tables": [
    { "title": "대학별 입시결과 범위 (1)", "columns": ["대학명","최대","중앙값","최소","평균"], "rows": [["서울대", 99.07, 96.5, 92.4, 96.18], ...] }
  ]
}
```

- `intro`: 상단 안내/설명 문단. 없으면 `[]`.
- `tables`: 0개 이상. 각 `{ title, columns, rows }`. `title`이 `null`이면 제목 없이 렌더.
- 인덱스 파일 `public/data/special/index.json`: `[{ key, sheet, tables, introLines }]` (건수 확인용).

## 탭별 추출 규격

키/시트/헤더행은 아래 확정값(엑셀 확인 완료).

### ① 안내필독 → key `notice`
- 표 없음(`tables: []`). B열(일부 C열) 텍스트를 위→아래 순서로 `intro`에 담는다. 빈 행은 스킵.
- 소제목 판정은 **추출 시** 하지 않는다. `SpecialTab`이 렌더 시 "숫자/★/공백으로 시작하지 않고 문장부호가 없는 짧은 줄"을 소제목으로 강조(굵게)한다. (예: `사용법-선행작업`, `사용법-검색시트`, `사용법-해석`)

### ② 백분위조견표 → key `johgyeon`
- 헤더 row 10, 데이터 row 11~136.
- `intro`: 상단 "시트 활용 정보" 병합셀 텍스트를 줄 단위로.
- 표 3개(가로 패널, 열 위치 확정):
  - `대학별 입시결과 범위 (1)`: 대학명=B, 최대=O, 중앙값=P, 최소=Q, 평균=R
  - `대학별 입시결과 범위 (2)`: 대학명=CY, 최대=DO, 중앙값=DP, 최소=DQ, 평균=DR
  - `의약학·교대 세부`: 대학명(세부)=FE, 계열=FG, 70%컷=FH, 정원=FI, 경쟁률=FJ, 충원율=FK
- 각 패널: 대학명 셀이 비면 그 행 스킵(패널마다 행 수가 다름). `#VALUE!` 등 오류 문자열은 빈칸("")으로.

### ③ 등급변환표 → key `gradeConv`
- `intro`: "시트 활용 정보"(6줄) + 상담 예시(3줄) + 산출 과정(3줄) 텍스트.
- 표:
  - `변환 결과(예시 성적 5등급 1.7)`: 교과조합 · 25%-75% 범위 · 변환 등급.
    행 = 4개 조합(전과목/국수영사과/국수영과/국수영사). 원본은 조합별 값이 한 데이터 행에 가로로 나열되어 있으므로, 이를 **조합당 한 행으로 세로 전개**한다.
  - `<표1> 5등급→9등급(샘플)`: 5등급 · 9등급 (⁝ 생략행 포함, 원본 그대로).
  - `<표2> 석차누적비(샘플)`: 5등급 · 5등급_석차 · 5등급_석차누적비 · 9등급 · 9등급_석차 · 9등급_석차누적비.
- 계산기는 만들지 않는다(정적). 원본에 캐시된 예시 성적(1.7) 기준 값을 그대로 표시.

### ④ 지원경향 → key `trend`
- 헤더 row 17, 데이터 row 18~615.
- `intro`: 현재 기준 1줄 — "기준: {대학}(row8 C) / {전형}(D) / {세부전형}(E) / {모집단위}(G)". 셀 참조로 조립.
- 표 2개(좌우 병렬):
  - `모집단위 기준`: 순번=A, 권역=B, 대학=C, 전형=D, 세부전형=E, 계열=F, 모집단위=G, 모집인원=H, 지원사례수=I, 합격사례수=J, 합격률(%)=K
  - `세부전형 기준`: 대학=M, 전형=N, 세부전형=O, 계열=P, 지원사례수=Q, 합격사례수=R, 합격률(%)=S
- 각 표: 그 표의 대학명(모집단위 기준은 C, 세부전형 기준은 M) 셀이 비면 행 스킵(두 표의 행 수가 다름). 합격률은 소수 → 표시 시 반올림은 렌더 단계에서.

## 컴포넌트 설계

### `DataTable` (신규, `src/ui/DataTable.tsx`)
- props: `columns: string[]`, `rows: (string|number|null)[][]`, `title?: string`.
- 순수 프레젠테이션. 기존 RefTable의 `<table>` 마크업/스타일을 그대로 이관.
- 숫자 셀은 오른쪽 정렬(기존 RefTable 동작 유지). 소수는 그대로 출력(반올림은 데이터 단계에서 하지 않음 — 원본 값 보존).

### `RefTable` (수정, `src/ui/RefTable.tsx`)
- 기존 fetch/로딩/에러 로직 유지. 표 렌더 부분만 `DataTable`을 호출하도록 교체.
- 외부 동작·출력은 변화 없음(리팩터링).

### `SpecialTab` (신규, `src/ui/SpecialTab.tsx`)
- props: `tabKey: string` (예 `notice`).
- `public/data/special/<key>.json` fetch. RefTable과 동일한 로딩/에러 패턴.
- 렌더: `intro` 문단(소제목 강조 규칙 적용) → 각 `tables[i]`를 `DataTable`로.
- 표가 0개면 문단만, 문단이 0개면 표만.

### `App.tsx` (수정)
- `SPECIAL_TABS = [{key:'notice',label:'안내필독'}, {key:'johgyeon',label:'백분위조견표'}, {key:'gradeConv',label:'등급변환표'}, {key:'trend',label:'지원경향'}]`.
- `TABS`에 판정 다음, 참고 탭 앞(또는 뒤) 위치에 삽입. 배치: 판정 → 참고 8탭 → **특수 4탭** → 합불 2탭. (안내필독은 성격상 맨 앞이 자연스러우나, 기존 순서 최소 변경을 위해 특수 묶음으로 참고 뒤에 배치.)
- `visitedTabs` lazy-mount 패턴 재사용: 방문한 특수 탭만 마운트.

## 추출 파이프라인

- `extract.py`에 `extract_special()` 추가. `__main__`에서 인자 `special`로 실행(`refs`/`hapbul`과 동형): `python scripts/extract.py special`.
- 4개 함수(또는 설정 주도) → `public/data/special/*.json` + `index.json`.

## 테스트 / 검증

- **추출 검증(수동)**: `python scripts/extract.py special` 실행 후 각 JSON의 `tables` 행 수·열 수와 첫 행 값이 원본과 일치하는지 눈으로 확인(예: 백분위조견표 (1) 첫 행 = 서울대 / 99.07 / 96.5 / 92.4 / 96.18).
- **컴포넌트 스모크 테스트(선택)**: 기존 테스트 인프라(vitest)는 엔진 위주라 UI 테스트는 없다. `DataTable` 분리가 RefTable 출력을 바꾸지 않음을 보장하기 위해, 기존 참고 탭 렌더가 회귀 없는지 dev 서버로 육안 확인.
- **빌드**: `npm run build` 에러 없음.
- **dev 육안 확인**: 4개 탭 전환 시 표/문단 정상, 기존 8개 참고 탭 회귀 없음, 콘솔 에러 없음.

## 비범위 (YAGNI)

- 등급변환기 계산 로직, 지원경향 대학/전형 선택 필터, 백분위조견표 차트 시각화 — 이번엔 만들지 않는다(정적 표만).
- 정렬/검색/페이지네이션 — 특수 탭에는 넣지 않는다(참고 탭과 동일하게 원본 순서 그대로). 지원경향(최대 ~598행)은 세로 스크롤로 충분.

## 열린 항목

- 백분위조견표 패널 (1)/(2)의 의미(인문/자연? 상·하위?)는 원본에 라벨이 없어 불명 → 중립 제목 `(1)`/`(2)`로 표기. 추후 사용자가 의미를 알면 제목만 교체.
